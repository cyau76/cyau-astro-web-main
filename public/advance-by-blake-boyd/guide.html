Advance Guide

Production documentation tools for live sound: patch sheets, stage plots, and run of show.

Quick Start

Create your first document in under a minute:

  1. Pick a tool. The home page shows three cards: Patch Sheet, Stage Plot, and Run of Show. Click one to open that tool's workspace.
  2. Create a document. Click "+ New" in the sidebar or "New Document" in the empty state. Give it a name in the toolbar at the top.
  3. Start editing. Fill in your channels, place your stage elements, or build your timeline. Everything auto-saves as you work.

That covers the basics. The sections below explain each tool in detail, plus exporting, importing, and keyboard shortcuts.

What is Advance?

In live sound, "the advance" is the process of sending technical documentation to a venue before a gig: patch lists, stage layouts, schedule cues. Advance is a browser-based tool suite for building those documents without accounts, subscriptions, or internet access after the initial page load.

Three tools, one workspace:

  • Patch Sheet: Input and output channel lists for your console. Track mic types, phantom power, connection routing, and notes per channel.
  • Stage Plot: A visual 2D layout of the stage. Drag instruments, monitors, risers, and personnel into position.
  • Run of Show: A cue-based timeline for production events. Auto-calculated start times, custom columns, section headers, and row highlighting.

All data stays in your browser's localStorage. Nothing is sent to a server. Export to JSON or PDF whenever you need a portable copy.

Navigation

The app header contains tabs for each tool. Click a tab to switch between Patch Sheets, Stage Plots, and Run of Show. The home icon returns you to the dashboard, which shows all three tools and your five most recent documents.

Each tool has a sidebar on the left listing your documents of that type, sorted by last edited. Click a document to open it, or click "+ New" to create one. The search bar at the top of the sidebar filters documents by name.

Patch Sheets

A patch sheet documents every audio channel on your console: what's plugged in, where it comes from, and how it's routed.

Metadata

The top section captures show-level information: venue, date, FOH engineer, monitor engineer, and general notes. These fields appear on PDF exports and help identify the document later.

Input Channels

Each row represents one input channel with the following columns:

  • Channel number: The console channel (editable, not auto-numbered, so you can skip or label channels as needed)
  • Name: Source name (e.g., "Kick In," "Vox Lead")
  • Mic Type: Dynamic, condenser, DI, ribbon, or wireless
  • Device: Specific mic or DI model
  • 48V: Phantom power toggle
  • Connection: Analog snake, Dante, or direct
  • Details: Connection specifics (snake channel, Dante route, etc.)
  • Notes: Anything else worth knowing

Click any cell to edit it inline. New rows appear at the bottom. Drag the handle on the left to reorder.

Output Channels

A separate tab for outputs (monitor mixes, aux sends, matrix outputs). Columns include channel number, name, source type, source details, destination type, destination gear, and notes.

Tip

Use the metadata fields even for informal gigs. When you re-open the document months later, "The Venue, Jan 15" is more useful than "Untitled Patch Sheet."

Stage Plots

A visual representation of what goes where on stage. The canvas shows a top-down view with "AUDIENCE" labeled at the bottom.

The Canvas

The canvas is a fixed-size area representing the stage. Choose from eight size presets (small to x-large, narrow or wide) using the dropdown above the canvas. The canvas can scroll horizontally on smaller screens.

Adding Elements

The element palette on the right contains 23 element types organized into four categories:

  • Instruments: Electric guitar, acoustic guitar, bass, drums, keyboard, piano, percussion, violin, horn, generic instrument
  • Audio: Microphone, monitor wedge, speaker, DI box, amp, headphones, mixer
  • People: Vocalist, musician, tech crew
  • Stage: Riser, table, chair

Click an element type to place it on the canvas. Each element appears with a default label that you can edit in the properties panel.

Moving and Resizing

Drag any element to reposition it. When selected, a resize handle appears at the bottom-right corner. Drag it to change the element's size. Enable grid snapping (the "Snap" checkbox above the canvas) to align elements to a 10-pixel grid.

Properties Panel

Select an element to see its properties on the right:

  • Label: Display name on the element
  • Position: X/Y coordinates (editable for precise placement)
  • Size: Width and height
  • Rotation: Angle in degrees
  • Color: Element background color
  • Show Label: Toggle label visibility
  • Duplicate and Delete buttons

Background Image

Upload an image (a floor plan, venue photo, or CAD drawing) as a background behind your elements. Adjust the opacity slider to control how much the background shows through. Click "Clear" to remove it.

Tip

Enable grid snap when placing multiple elements so they line up naturally. Disable it for final fine-tuning.

Run of Show

A timeline for production events: sound checks, set changes, performances, intermissions. Each row is a cue with timing information and department-specific notes.

Built-in Columns

Every run of show includes these columns:

  • #: Item number (auto-increments but editable)
  • Start: Start time in HH:MM:SS format
  • Duration: How long in MM:SS format
  • Audio: Sound cues and notes
  • Video: Video/projection cues
  • Lights: Lighting cues
  • Notes: Production notes

Custom Columns

Click "Add Column" to create additional columns for your specific needs (e.g., "Backline," "Pyro," "MC Script"). Custom columns support text, number, and time types. Click the column header to rename or delete it.

Section Headers

Click "Add Header" to insert a section divider (e.g., "Sound Check," "Doors Open," "Headliner Set"). Headers span the full width of the table and help visually organize a long timeline.

Row Highlighting

Click the color dot on any row to highlight it with one of nine colors: red, amber, green, blue, purple, pink, cyan, or lime. Useful for flagging critical cues, color-coding by department, or marking tentative items.

Reordering

Drag the grip handle on the left of any row to reorder items. The start times do not auto-cascade (each row's start time is independent), so adjust manually after reordering.

Tip

Use section headers to break long shows into logical blocks: load-in, sound check, doors, set 1, changeover, set 2, load-out. This makes the document scannable during the show.

Saving and Auto-Save

Documents auto-save to your browser's localStorage as you edit, with a short delay to batch rapid changes. A status indicator in the toolbar shows "Saving..." during the delay and "Saved" when the save completes.

Your documents persist between browser sessions. They survive page refreshes and browser restarts. However, clearing your browser data or using private/incognito mode will erase them.

Tip

Export important documents to JSON regularly. Local storage is convenient but not a backup. A JSON file on your drive is.

Exporting

The export menu (in the toolbar when a document is open) offers three formats:

JSON

A full data export that can be re-imported later. This is the best option for backups, sharing with collaborators, or moving data between devices. The file contains all document data including metadata, channel lists, element positions, and timeline items.

PDF

A print-ready document. Patch sheets and run of show export as formatted tables with headers and metadata. Stage plots export as an image snapshot of the canvas. PDFs auto-orient based on content (landscape for wide stage plots, portrait for tables).

Import

Click "Import" in the export menu to load a previously exported JSON file. The importer validates the file format and document type before creating a new document with the imported data.

Undo and Redo

Every editing action can be undone. Use the toolbar buttons or keyboard shortcuts. The undo/redo buttons gray out when there's no history to traverse.

Undo history is per-document and resets when you switch documents or reload the page. It tracks all state changes: adding/removing rows, editing cells, moving elements, changing properties.

Keyboard Shortcuts

UndoCtrl/Cmd + Z
RedoCtrl/Cmd + Y

Privacy

Advance runs entirely in your browser. Your documents, channel lists, stage layouts, and timelines are:

  • Never sent to any server
  • Stored only in your browser's localStorage
  • Deletable at any time (click the trash icon on any document)

Exported files stay on your device. JSON exports contain only the data you entered. PDF exports are generated client-side and never uploaded.

Tips

  • Name your documents well. "Spring Concert 2026 - Main Stage" is more useful than "Untitled Patch Sheet" when you have a dozen documents in the sidebar.
  • Use all three tools together. A complete advance package for a venue includes a patch sheet, a stage plot, and a run of show. Export all three as PDFs and send them in one email.
  • Export before clearing browser data. JSON exports round-trip perfectly. If you're switching browsers or devices, export your documents first and import them on the other end.
  • Start from templates. Create a base document for your typical setup, export the JSON, then import it for each new gig and modify from there.
  • Grid snap for clean plots. Stage plots look more professional when elements are aligned. Enable snap, place your elements roughly, and they'll settle onto the grid.
  • Color-code your run of show. Establish a consistent color scheme across shows: green for go cues, red for hold points, amber for contingencies.
